# Articles & Blog Posts

> **Metodikk-fil:** Faktiske verdier (tone, keywords) hentes fra `./marketing/BRAND.md` og `./marketing/CONTENT-RULES.md`.

Artikler er grunnsteinen i content marketing. De bygger autoritet, driver organisk trafikk, og posisjonerer merkevaren som ekspert.

---

## Artikkel-arketyper

### 1. Thought Leadership

**Formål:** Etablere ekspertise og dele meninger

**Struktur:**
```
1. Hook: Kontroversiell påstand eller observasjon
2. Kontekst: Hvorfor dette betyr noe nå
3. Din posisjon: Tydelig standpunkt med begrunnelse
4. Bevis: Data, eksempler, case studies
5. Implikasjoner: Hva dette betyr for leseren
6. CTA: Hva bør leseren gjøre?
```

**Eksempel-titler:**
- "Hvorfor [vanlig praksis] er feil tilnærming"
- "[Trend] vil endre [bransje] - her er hvorfor"
- "Det ingen snakker om: [innsikt]"

**Tips:**
- Ta et tydelig standpunkt (fra BRAND.md Communication Principles)
- Bruk førsteperson ("Vi mener...", "Vår erfaring viser...")
- Inkluder motargumenter og adresser dem

---

### 2. How-To / Tutorial

**Formål:** Lære leseren noe praktisk

**Struktur:**
```
1. Hook: Hva leseren vil kunne gjøre etter å ha lest
2. Forutsetninger: Hva trengs før start?
3. Oversikt: Kort oppsummering av stegene
4. Steg-for-steg: Nummererte, handlingsbare steg
5. Verifisering: Hvordan vet de at det fungerer?
6. Vanlige feil: Hva går ofte galt?
7. Neste steg: Hva kan de lære mer?
```

**Eksempel-titler:**
- "Hvordan [oppnå resultat] på [tid]"
- "Den komplette guiden til [emne]"
- "[X] steg for å [resultat]"

**Tips:**
- Ett steg = én handling
- Inkluder skjermbilder/eksempler
- Høy AEO-verdi - optimaliser for HowTo schema

---

### 3. Nyhetskommentar / Industry Analysis

**Formål:** Kommentere på hendelser og trender

**Struktur:**
```
1. Hook: Hva skjedde og hvorfor det betyr noe
2. Bakgrunn: Kort kontekst for de som ikke fulgte med
3. Analyse: Din tolkning og innsikt
4. Implikasjoner: Hva betyr dette for leseren?
5. Anbefaling: Hva bør de gjøre?
```

**Eksempel-titler:**
- "Hva [hendelse] betyr for [bransje]"
- "[Selskap] sin nye strategi: Vår analyse"
- "3 ting vi lærte av [hendelse]"

**Tips:**
- Publiser raskt (timeliness matters)
- Link til primærkilder
- Gi unik vinkling, ikke bare oppsummering

---

### 4. Listicle

**Formål:** Kuratere og rangere alternativer

**Struktur:**
```
1. Hook: Hvorfor denne listen er verdifull
2. Kriterier: Hvordan ble listen laget?
3. TL;DR: Quick picks for de som har det travelt
4. Listen: Hvert element med beskrivelse
5. Konklusjon: Overordnet anbefaling
```

**Eksempel-titler:**
- "[X] beste [ting] for [målgruppe] i [år]"
- "De viktigste [ting] du trenger å vite om [emne]"
- "[X] feil å unngå når du [handling]"

**Tips:**
- Unngå padding (kvalitet > kvantitet)
- 5-10 elementer er sweet spot
- Inkluder "Beste for..." per element

---

### 5. Case Study

**Formål:** Vise resultater gjennom ekte historier

**Struktur:**
```
1. Hook: Resultat-tall eller transformasjon
2. Kunden: Hvem er de? (med tillatelse)
3. Utfordringen: Hva var problemet?
4. Løsningen: Hva gjorde dere?
5. Implementering: Hvordan fungerte det i praksis?
6. Resultater: Konkrete tall og outcomes
7. Sitat: Kundens egne ord
8. Lærdommer: Hva kan andre ta med seg?
```

**Eksempel-titler:**
- "Hvordan [kunde] oppnådde [resultat] med [løsning]"
- "Fra [før] til [etter]: [kunde] sin historie"
- "[Kunde]: [X]% økning i [metrikk]"

**Tips:**
- Spesifikke tall > vage påstander
- Inkluder timeline
- Vis prosessen, ikke bare resultatet

---

### 6. Sammenligning

**Formål:** Hjelpe leseren velge mellom alternativer

**Struktur:**
```
1. Hook: "Hvilket alternativ er best for deg?"
2. TL;DR: Quick verdict
3. Sammenligningstabell: Side-by-side
4. Alternativ A: Dypdykk
5. Alternativ B: Dypdykk
6. Når velge A vs B: Beslutningshjelp
7. Konklusjon: Klar anbefaling
```

**Eksempel-titler:**
- "[A] vs [B]: Hvilken er best for [målgruppe]?"
- "Den ærlige sammenligningen: [A] mot [B]"
- "[A] eller [B]? Her er svaret"

**Tips:**
- Vær ærlig om styrker/svakheter
- Inkluder "Velg A hvis... Velg B hvis..."
- Oppdater regelmessig (priser endres)

## Hook-teknikker (Første 100 ord)

### Den direkte åpningen
Svar på spørsmålet umiddelbart.

```
Det beste CRM-systemet for små bedrifter er [X]. Her er hvorfor,
og når du bør vurdere alternativer.
```

### Kontroversiell påstand
Utfordre konvensjonell visdom.

```
De fleste content marketing-strategier feiler. Ikke fordi innholdet
er dårlig, men fordi det løser feil problem.
```

### Spesifikt tall
Bruk data for å fange oppmerksomhet.

```
Vi analyserte 500 landingssider. 73% av dem gjorde samme feil.
Her er hva de overså.
```

### Leseren i fokus
Start med deres situasjon.

```
Du har prøvd alt: SEO, annonser, sosiale medier. Trafikken kommer,
men konverteringene uteblir. Problemet er sannsynligvis ikke kanalen.
```

### Historien
Start med en konkret situasjon.

```
Forrige uke spurte en kunde: "Hvorfor rangerer vi ikke på noe?"
Svaret overrasket henne.
```

## Skannbarhets-sjekkliste

### Heading-hierarki
- [ ] Én H1 (tittelen)
- [ ] H2 for hovedseksjoner
- [ ] H3 for underseksjoner
- [ ] Aldri hopp over nivåer (H2 → H4 er feil)

### Avsnitt
- [ ] 2-4 setninger per avsnitt
- [ ] Én idé per avsnitt
- [ ] Visuelt skille mellom avsnitt

### Lister
- [ ] Bullet points for uordnet informasjon
- [ ] Nummerering for sekvenser/prioritering
- [ ] Maks 7 punkter per liste (kognitivt tak)

### Formattering
- [ ] Bold for nøkkelbegreper (sparsomt)
- [ ] Kursiv for sitater eller fremhevelse
- [ ] Kodeblokker for teknisk innhold
- [ ] Callout-bokser for viktig info

### Visuelle elementer
- [ ] Bilder med beskrivende alt-tekst
- [ ] Tabeller for sammenligning
- [ ] Diagrammer for prosesser
- [ ] Screenshots for tutorials

## Intern lenke-strategi

### Hvor mange?
- **Minimum:** 3 interne lenker per artikkel
- **Anbefalt:** 5-8 for lange artikler (2000+ ord)
- **For mye:** Mer enn 1 lenke per 100 ord

### Hvor plassere?
| Plassering | Type lenke |
|------------|------------|
| Intro | Lenke til pillar page |
| Hovedinnhold | Lenke til relaterte artikler |
| Konklusjon | Lenke til neste steg/CTA |

### Ankertekst-regler
**Godt:**
- "Les mer om [emne] i vår guide"
- "Se hvordan du [handling] her"
- "[Emne] forklart i detalj"

**Dårlig:**
- "Klikk her"
- "Les mer"
- "Her"

### Pillar/Cluster-modell
```
Pillar Page (hovedemne)
├── Cluster Article 1 → lenker til pillar
├── Cluster Article 2 → lenker til pillar
└── Cluster Article 3 → lenker til pillar + hverandre
```

## AEO-optimalisering for artikler

### Answer blocks (40-60 ord)
Skriv svar som kan stå alene og siteres av AI.

**Eksempel:**
```
Content marketing er en strategisk tilnærming fokusert på å skape
og distribuere verdifullt, relevant innhold for å tiltrekke og
beholde en tydelig definert målgruppe. Målet er å drive
lønnsomme kundehandlinger gjennom å bygge tillit og autoritet
over tid, i stedet for direkte salg.
```

### FAQ-seksjon
Inkluder 3-5 spørsmål i FAQ-format med schema-ready struktur.

```markdown
## FAQ

### Hva er [emne]?

[Direkte svar på 40-60 ord]

### Hvorfor er [emne] viktig?

[Direkte svar på 40-60 ord]

### Hvordan kommer jeg i gang med [emne]?

[3-5 nummererte steg]
```

### Strukturformater AI elsker
| Format | Bruk for |
|--------|----------|
| Definisjon | "Hva er X?" |
| Liste | "Hvilke X finnes?" |
| Steg-for-steg | "Hvordan gjør jeg X?" |
| Sammenligning | "X vs Y" |
| Tabell | "Forskjeller mellom X og Y" |

## Artikkel-kvalitetssjekk

### Før du publiserer

**Struktur:**
- [ ] Hook fanger oppmerksomhet i første 2 setninger
- [ ] Intent løst i første 100 ord
- [ ] TL;DR/Key takeaways inkludert
- [ ] Korrekt H2/H3 hierarki
- [ ] 3-5 interne lenker
- [ ] FAQ-seksjon med 3-5 spørsmål

**Voice:**
- [ ] Matcher BRAND.md Tone of Voice
- [ ] Ingen Words We Avoid
- [ ] Følger CONTENT-RULES.md voice-dimensjoner
- [ ] Leses naturlig høyt

**SEO/AEO:**
- [ ] Primary keyword i tittel
- [ ] Primary keyword i H1
- [ ] Primary keyword i første 100 ord
- [ ] Answer blocks på 40-60 ord
- [ ] Alt-tekst på alle bilder

**Kvalitet:**
- [ ] Spesifikke tall og eksempler
- [ ] Kilder med attribusjon

- [ ] "Ville jeg lest dette selv?"-test bestått

## Vanlige feil å unngå

### Generiske åpninger
**Dårlig:** "I dagens digitale verden er content marketing viktig."
**Godt:** "73% av markedsførere sier content marketing er deres viktigste kanal. Her er hvorfor."

### For lange introer
**Dårlig:** 300 ord bakgrunn før hovedpoenget
**Godt:** Hovedpoeng i første 100 ord, kontekst etterpå

### Ingen tydelig struktur
**Dårlig:** Lange avsnitt uten overskrifter
**Godt:** Klare H2/H3, korte avsnitt, bullet points

### Vag informasjon
**Dårlig:** "Mange bedrifter opplever vekst med denne metoden."
**Godt:** "Bedrifter som bruker denne metoden ser i snitt 34% økning i leads."

### Manglende CTA
**Dårlig:** Artikkel som bare... slutter.
**Godt:** Tydelig neste steg for leseren.
